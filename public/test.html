Option A — Simple Center (추천)

React에서 상태 관리 잘하는 법

react frontend state

상태 관리는 React 앱에서 가장 중요한 설계 결정 중 하나입니다. 잘못된 선택은 컴포넌트 트리 전체를 복잡하게 만들고, 디버깅을 어렵게 만들 수 있습니다.

로컬 상태 vs 전역 상태

모든 상태를 전역으로 올리려는 충동을 참아야 합니다. 상태는 그것을 필요로 하는 컴포넌트에 최대한 가깝게 유지하세요.

  • UI 상태 (모달 열림/닫힘, 탭 선택) → 로컬 useState
  • 서버 데이터 → React Query 또는 SWR
  • 공유 앱 상태 → Context API 또는 Zustand

Context API의 함정

Context는 강력하지만 모든 컨슈머를 리렌더링시킵니다. 값이 자주 바뀌는 상태에는 사용을 피하세요.

// 잘못된 예
const AppContext = createContext({ user, theme, notifications })

// 올바른 예: 목적별 분리
const UserContext = createContext(user)
const ThemeContext = createContext(theme)

결론

상태 관리의 황금률은 "필요한 만큼만"입니다. 먼저 로컬 상태로 시작하고, 실제로 공유가 필요해질 때 올리세요. 과도한 추상화는 오히려 코드를 복잡하게 만듭니다.
